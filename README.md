# โปรเจกต์: เกมน้ำเต้าปูปลา (Console-Based, Functional Programming, TypeScript)

โปรเจกต์นี้เป็นระบบจำลองการเล่นเกม **"น้ำเต้าปูปลา"** แบบควบคุมผ่านคอนโซล (CLI) ที่ได้รับการพัฒนาภายใต้กรอบแนวคิด **Functional Programming (FP) อย่างเคร่งครัด** โดยใช้ภาษา TypeScript และรันบน Node.js เพื่อจุดประสงค์ในการแยกตรรกะทางธุรกิจ (Core Logic) ออกจากส่วนติดต่อเชื่อมโยงภายนอก (Impure Shell/Effects) อย่างชัดเจน

---

## 1. วิธีติดตั้งและรันโปรเจกต์

โปรเจกต์นี้ได้รับการทดสอบบนสภาพแวดล้อม Node.js โดยสามารถดำเนินการตามขั้นตอนด้านล่างนี้:

### การติดตั้ง Dependencies
เปิด Terminal ในโฟลเดอร์หลักของโปรเจกต์แล้วรันคำสั่ง:
```bash
npm install
```

### การเริ่มเล่นเกม (Start CLI Game)
รันคอมมานด์ไลน์เพื่อเล่นเกมจำลองในสภาวะจริง:
```bash
npm start
```

## 2. โครงสร้างโปรเจกต์ (Folder Structure) และเหตุผลทางสถาปัตยกรรม

โปรเจกต์นี้ใช้สถาปัตยกรรม **Single File + Comment Section** โดยรวมซอร์สโค้ดทั้งหมดไว้ในไฟล์เดียวคือ `namtao.ts` เพื่อความสะดวกในการส่งตรวจและรันด้วยคำสั่งเดียว แต่ยังคงแยกส่วน Pure Core Logic และ Impure Shell อย่างชัดเจนด้วย Comment Section ภายในไฟล์:

```text
namtao/
├── namtao.ts             # ซอร์สโค้ดทั้งหมด (Types, Pure Core, Impure Effects, CLI Loop, Bootstrap)
├── package.json          # กำหนดรายละเอียด dependencies และสคริปต์รัน
├── tsconfig.json         # ตั้งค่า Compiler Options ของ TypeScript
└── game.log              # ไฟล์บันทึกประวัติการเล่น (สร้างขึ้นหลังเล่นเกม)
```

### โครงสร้างภายในไฟล์ `namtao.ts` (แบ่งตาม Comment Section)
```text
// Section 0: TYPES & DOMAIN MODELS         — ประเภทข้อมูลทั้งหมด (GameSymbol, DiceResult, Bet, ...)
// Section 1: PURE CORE LOGIC                — pipe/compose, กติกาเกม, ไปป์ไลน์, parseBetInput
// Section 2: IMPURE EFFECTS                 — rollDice, generateRoundBonus, writeLogToFile, promptUser
// Section 3: MAIN GAME LOOP (CLI)           — formatBetMap, formatRoundBonus, gameLoop
// Section 4: BOOTSTRAP                      — main, entry point
```

### ทำไมจึงเลือกรวมเป็นไฟล์เดียว
โปรเจกต์ถูกพัฒนาในช่วงแรกแบบแยกโมดูลหลายไฟล์ (`src/core/`, `src/effects/`, `src/types/`, `src/utils/`) เพื่อฝึกการแยกโครงสร้าง หลังจากพัฒนาครบทุกฟีเจอร์และ Unit Tests ผ่านทั้งหมดแล้ว จึงทำการ Refactor รวมเป็นไฟล์เดียวเพื่อ:
1. **ส่งตรวจง่าย**: อาจารย์สามารถเปิดไฟล์เดียวและอ่านโค้ดตั้งแต่ต้นจนจบได้ทันที
2. **รันง่าย**: ใช้คำสั่ง `npm start` เพียงคำสั่งเดียว ไม่ต้องกังวลเรื่อง path ของโมดูล
3. **ยังคงแยก Pure/Impure ชัดเจน**: แม้จะอยู่ในไฟล์เดียว แต่ยังคงแยกส่วน Pure Core และ Impure Shell ด้วย Comment Section อย่างเคร่งครัด ทุกฟังก์ชันยังมี tag `// PURE` หรือ `// IMPURE` กำกับครบทุกตัว

### หลักการ Functional Core, Imperative Shell ที่ยังคงไว้
1. **Testability**: ฟังก์ชัน Pure ทั้งหมดสามารถทดสอบได้อย่างแม่นยำ 100% โดยไม่ต้อง Mock เพราะไม่มีการอ้างอิงกับเวลา, ระบบสุ่ม หรือตัวแปรภายนอก
2. **Determinism**: งานที่ไม่แน่นอน (เช่น การทอยเต๋า, การสุ่มโบนัส) ถูกแยกไว้ใน Section 2 (Impure Effects) และผลลัพธ์ถูกฉีด (Inject) เข้าสู่ Pure Pipeline เสมอ
3. **Decoupling**: โค้ดส่วนคำนวณเงินรางวัลไม่ขึ้นตรงกับเครื่องมือรับค่าทางหน้าจอ สามารถนำไปใช้ซ้ำบนแพลตฟอร์มอื่นได้ง่าย

## 3. แนวคิด Functional Programming ที่นำมาใช้

### 3.1 Pure Function & Tagging
ทุกฟังก์ชันในโปรเจกต์จะต้องมีการประกาศคุณลักษณะอย่างชัดเจนผ่านแท็กคำอธิบายเหนือลายเซ็นต์ฟังก์ชัน:
- `// PURE` สำหรับฟังก์ชันที่ไม่มี Side Effect และทำงานแบบ Deterministic
- `// IMPURE` สำหรับฟังก์ชันที่ส่งผลกระทบต่อภายนอก อ่านสถานะเครื่อง หรือมีความสุ่ม

ตัวอย่าง:
```typescript
// PURE
export function calculateBasePayout(betAmount: number, matchCount: number): number {
  return matchCount > 0 ? betAmount * (matchCount + 1) : 0;
}
```

### 3.2 Immutability
เรากำหนดโครงสร้างข้อมูลเป็น `readonly` เสมอ เพื่อป้องกันการแก้ไขข้อมูลเดิมโดยตรง (Mutation) แต่จะใช้วิธีสร้างและส่งคืน State ชุดใหม่แทน:
```typescript
export type StreakState = {
  readonly consecutiveWins: number;
  readonly bonusTimeActive: boolean;
  readonly bonusTimeMultiplier: number;
};
```

### 3.3 Function Composition (`pipe` / `compose`)
เราได้เขียนฟังก์ชันจัดเรียงไปป์ไลน์แบบกำหนดเองขึ้นมาใน Section 1 ของ [`namtao.ts`](file:///d:/work/Codex/namtao/namtao.ts) เพื่อร้อยเรียงข้อมูลแบบเป็นลำดับ:
```typescript
// PURE
export function pipe(value: any, ...fns: Function[]): any {
  return fns.reduce((acc, fn) => fn(acc), value);
}
```

ฟังก์ชันไปป์ไลน์จะร้อยเรียงขั้นตอนการคำนวณผลลัพธ์ของรอบ ดังนี้:
```typescript
// PURE
export function runRoundPipeline(input: RoundInput): RoundResult {
  return pipe(
    initRoundContext(input),
    matchResultStep,
    calculateBasePayoutStep,
    resolveMultiplierSourceStep,
    applyFinalMultiplierStep,
    updateStreakStateStep,
    updateStateStep
  );
}
```

### 3.4 Dependency Injection ของ Side Effects
แทนที่จะเรียกใช้ `Math.random()` ภายใน Core Logic โดยตรง เราได้แยกมันไว้ใน Section 2 (Impure Effects) ของ `namtao.ts` แล้วนำผลลัพธ์ที่ได้ **ฉีด (Inject)** เข้าสู่ไปป์ไลน์หลักผ่าน Arguments แทน:
```typescript
// ใน CLI loop (Impure Shell)
const dice = rollDice(); // เรียกฟังก์ชันสุ่ม (Impure)
const result = runRoundPipeline({
  dice,                  // ฉีดผลการสุ่มเข้าไปยัง Pure Core Logic
  ...
});
```

## 6. กติกาและระบบพิเศษของเกม

### 6.1 กติกาพื้นฐาน
- สัญลักษณ์ในลูกเต๋ามี 6 หน้า ได้แก่: **น้ำเต้า, ปู, ปลา, ไก่, เสือ, กุ้ง**
- ทอยลูกเต๋า 3 ลูกพร้อมกัน
- ผู้เล่นชนะในรอบนั้น หากมีอย่างน้อยหนึ่งช่องที่แทงแล้วตรงกับผลเต๋า
- อัตราจ่ายปกติ: ตรง 1 ลูก ได้ x1 ของยอดแทง, ตรง 2 ลูก ได้ x2, ตรง 3 ลูก ได้ x3 (พร้อมคืนทุนสำหรับช่องที่ถูก)
- หากไม่ถูกช่องใด ๆ เลย จะเสียเงินเดิมพันในช่องนั้นทั้งหมด

### 6.2 ระบบที่ 1: Random Round Bonus (โบนัสสุ่มประจำรอบ)
- ทำงานเฉพาะตอนที่ผู้เล่น **ไม่ได้อยู่ในช่วง Bonus Time** เท่านั้น
- ก่อนเริ่มเดิมพัน ระบบจะสุ่มสัญลักษณ์ 0 - 3 ตัวเพื่อมอบโบนัสตัวคูณพิเศษ โดยสุ่มตัวคูณจาก Pool ที่กำหนด: **x2, x4, x6, x8, x10** เท่านั้น
- ตัวคูณนี้จะคูณเข้ากับเงินรางวัลปกติ (รวมทุน) ของสัญลักษณ์ที่ถูกเดิมพันเท่านั้น

### 6.3 ระบบที่ 2: Bonus Time (โบนัสสตรีคชนะติดต่อกัน)
- หากผู้เล่น **ชนะ** ติดต่อกันครบ 3 ตา (มีช่องที่แทงถูกอย่างน้อย 1 ช่อง 3 รอบติดกัน) จะเข้าสู่ **Bonus Time** ในรอบถัดไป
- เมื่ออยู่ใน Bonus Time:
  - ระบบจะ**ไม่มีการสุ่มโบนัสประจำรอบ (Random Round Bonus)** (ห้ามใช้ระบบทับซ้อนกัน)
  - ทุกช่องที่ผู้เล่นแทงถูกในตานั้นจะได้รับการคูณเพิ่ม **x2 ทั้งกระดาน** โดยอัตโนมัติ
  - หากเล่นชนะต่อเนื่อง ตัวคูณทวีคูณจะขยับขึ้นทีละรอบ: **x2 → x3 → x4 → x5 → x6** (สูงสุดและคงที่ที่ x6)
  - หาก **แพ้** เพียงครั้งเดียว โหมด Bonus Time จะยกเลิกทันที ยอดสตรีคกลับเป็น 0 และกลับสู่การเล่นโหมดปกติ

## 7. ตัวอย่างการเล่นเกมจริง (Gameplay Simulation)

ต่อไปนี้เป็นตัวอย่างสถานการณ์จำลอง 3 รอบของการเล่น:

### รอบที่ 1: โหมดปกติ ได้รับโบนัสสุ่มประจำรอบ
- **สถานะเริ่มต้น**: ยอดเงินคงเหลือ = 1,000 บาท, ชนะติดต่อกัน = 0, ไม่ได้อยู่ใน Bonus Time
- **โบนัสสุ่มประจำรอบที่เกิดขึ้น**: `เสือ x4` และ `กุ้ง x10`
- **ผู้เล่นแทง**: `เสือ 100 บาท` และ `ปลา 100 บาท` (รวมวางเดิมพัน 200 บาท)
- **ผลการทอยเต๋า**: `[เสือ] [ปลา] [ไก่]`
- **ผลลัพธ์**:
  - `เสือ`: แทงถูก 1 ลูก -> ฐานจ่ายปกติ = 200 (ชนะ 100 + คืนทุน 100) -> ได้รับโบนัส `เสือ x4` -> ได้รับคืนจริง = 200 * 4 = 800 บาท
  - `ปลา`: แทงถูก 1 ลูก -> ฐานจ่ายปกติ = 200 -> ไม่มีโบนัสประจำรอบ -> ได้รับคืนจริง = 200 บาท
  - **สรุป**: แทงรวม 200 บาท ได้รับคืนรวม 1,000 บาท (กำไรสุทธิ +800 บาท)
  - **สถานะปลายรอบ**: ยอดเงินคงเหลือ = 1,600 บาท, สตรีคชนะ = 1

### รอบที่ 2: ปลดล็อกเข้าสู่โหมด Bonus Time
- **สถานะเริ่มต้น**: ยอดเงินคงเหลือ = 1,600 บาท, ชนะติดต่อกัน = 2 (ชนะมาจากรอบก่อนหน้า)
- **ผู้เล่นแทง**: `กุ้ง 100 บาท` (รวมวางเดิมพัน 100 บาท)
- **ผลการทอยเต๋า**: `[กุ้ง] [ไก่] [ปู]` (ชนะรอบนี้)
- **สถานะปลายรอบ**: ชนะติดต่อกันสะสมเป็น 3 ตา -> ปลดล็อก **Bonus Time** ในตาถัดไป
- **ยอดเงินใหม่**: 1,700 บาท

### รอบที่ 3: เล่นอยู่ในช่วง Bonus Time
- **สถานะเริ่มต้น**: ยอดเงินคงเหลือ = 1,700 บาท, โหมด Bonus Time ทำงาน (ตัวคูณทั้งกระดาน x2)
- **โบนัสสุ่มประจำรอบ**: *ไม่มีการสร้างโบนัสสุ่มสัญลักษณ์* เนื่องจากกติกาห้ามซ้อนกัน
- **ผู้เล่นแทง**: `ปลา 100 บาท` (รวมวางเดิมพัน 100 บาท)
- **ผลการทอยเต๋า**: `[ปลา] [ปลา] [น้ำเต้า]`
- **ผลลัพธ์**:
  - `ปลา`: แทงถูก 2 ลูก -> ฐานจ่ายปกติ = 300 (ชนะ 200 + คืนทุน 100) -> ได้รับผลคูณทั้งกระดาน x2 -> ได้รับเงินคืนจริง = 300 * 2 = 600 บาท
  - **สรุป**: แทงรวม 100 บาท ได้รับคืนรวม 600 บาท (กำไรสุทธิ +500 บาท)
  - **สถานะปลายรอบ**: ยอดเงินคงเหลือ = 2,200 บาท, สตรีคชนะ = 4, Bonus Time ขยับระดับตัวคูณขึ้นเป็น x3 สำหรับรอบต่อไป

## 8. แผนภูมิขั้นตอนการทำงานหลัก (Game Pipeline Diagram)

ไปป์ไลน์แสดงขั้นตอนการไหลเวียนของสถานะและการคำนวณภายในรอบการเล่นเกม:

```text
[ CLI Impure Shell / Effects ]
              │
              ▼
1. สุ่มโบนัสประจำรอบ (generateRoundBonus) หรือ ตรวจพบ Bonus Time
              │
              ▼
2. รับข้อมูลการเดิมพันจากหน้าคอนโซล (promptUser)
              │
              ▼
3. ตรวจสอบความถูกต้องและแปลงอินพุต (parseBetInput)
              │
              ▼
4. ทอยลูกเต๋า 3 ลูก (rollDice)
              │
              ▼
   ──────────────────────────────────────────────────────────
   [ Pure Pipeline - runRoundPipeline (fp.pipe) ]
   
   5. [ initRoundContext ]
      สร้างแบบจำลองสภาวะแวดล้อมตั้งต้นสำหรับใช้วิเคราะห์ผล
              │
              ▼
   6. [ matchResultStep ]
      คำนวณจำนวนหน้าเต๋าที่จับคู่ตรงกับช่องเดิมพันของผู้เล่น
              │
              ▼
   7. [ calculateBasePayoutStep ]
      คำนวณเงินรางวัลพื้นฐานแบบปกติของแต่ละหน้า (bet * (matchCount + 1))
              │
              ▼
   8. [ resolveMultiplierSourceStep ]
      ประเมินว่าตัวคูณของสัญลักษณ์มาจาก Bonus Time หรือ Random Round Bonus
              │
              ▼
   9. [ applyFinalMultiplierStep ]
      นำตัวคูณไปทวีคูณเข้ากับฐานการเงินรางวัลเดิม
              │
              ▼
  10. [ updateStreakStateStep ]
      ประเมินผล แพ้/ชนะ ของรอบ แล้วทำการคิดค้นระดับ Streak ใหม่
              │
              ▼
  11. [ updateStateStep ]
      หักล้างยอดคงเหลือสะสม และสังเคราะห์ข้อมูลประวัติเป็น RoundResult
   ──────────────────────────────────────────────────────────
              │
              ▼
12. บันทึกข้อมูลและบันทึกประวัติการเล่น (writeLogToFile)
              │
              ▼
13. แสดงผลหน้าจอพร้อมเริ่มรอบการทำงานถัดไป (gameLoop - Recurse)
```
